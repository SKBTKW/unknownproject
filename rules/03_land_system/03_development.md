# 03. 開発（ニコイチ）ルール

## 基本ルール
- **同じベース地形同士**でのみ開発可能（平地＋平地、丘陵＋丘陵など）
- 開発コスト: 🔥を**2消費**（通常の土地配置が1なので、+1の追加コスト）
- 開発すると**レアリティが1段階上昇**（C → UC → R → L）し、外見も豪華になる

## 特性の扱い
開発時、対象の2土地に付与されている特性を以下のように処理する：

- 特性が**2つ以下**の場合 → そのまま引き継ぐ
- 特性が**3つ以上**の場合 → プレイヤーが最大**2つまで選択**して引き継ぎ、残りは破棄
- **同じ特性が重複**した場合 → ボーナスが加算される（例: 果実×2 → より強い食料ボーナス）
- 森・密林・砂漠などの**同一属性**を開発する場合 → 特性効果にボーナスが付き、重複した属性を1つ破棄できる（強化扱い）

## 川の特殊ルール
川を特性に持つ土地は**開発自体が不可**。  
これは「川という強力な特性を無制限に強化できない」ようにするための制限。

## 設計意図
開発は「場の圧縮」と「特性の強化」の両方を兼ねる重要なアクション。
コストをやや重めに設定することで、安易に連発できないようにし、取捨選択のジレンマを生む。

## 開発による視覚表現

土地を開発した際の視覚的な変化についても、以下の方向性を採用する。

### 基本方針
- 開発回数に応じて**カードフレームを変化**させる
- 正確な開発回数や強化内容は**ツールチップ**で確認できるようにする
- テキストだけで「平地+1」などと表示するのではなく、視覚的に「土地が育った」と実感できる表現を目指す

### フレーム変化の方向性（案）
- **未開発（0回）**: 粗い木枠・土っぽい質感
- **開発1回**: 木枠に金属の縁取りが追加され、少し上質になる
- **開発2回**: 銀色の金属枠がメインになる
- **開発3回**: 金色の枠＋炎の模様が浮かび上がる
- **開発4回以上**: 金枠＋強く輝く炎のオーラや粒子エフェクト

### ツールチップでの表示例
カードにカーソルを合わせた際に、以下のような情報を表示する：
- 現在のベース地形と開発回数
- 各リソースの産出量
- 付与されている特性
- 開発による強化値（例：開発による強化 +12）

### 目的
- 土地を開発するたびに視覚的に変化することで、「土地を育てる」という行為の実感を強める
- 一目でその土地がどれだけ強化されているかを把握できるようにする
- 開発という行為に達成感と満足感を持たせる
