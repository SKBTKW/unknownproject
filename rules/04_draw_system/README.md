# 04. ドローシステム

## 基本思想

「場の状況によってドローをある程度コントロールできる」を最重要視する。
MTGのような完全な土地事故を避けつつ、プレイヤーの選択が次のドローに影響を与える設計を目指す。

## 6つのドローカテゴリ

|カテゴリ|優先度|主な内容|場の影響を受けやすいか|
|----------|------|----------|------------------------|
|🧭 土地発見|高|新しいベース土地を獲得|非常に受けやすい|
|🍎 土地特性付与|高|属性やボーナス資源を付与|非常に受けやすい|
|☠️ 危機・災害|低|自然災害・小規模襲撃|やや受けやすい（レガシーで低減）|
|👥 人間・社会・士気|中|イベント、クエスト、ボーナス|やや受けやすい|
|✨ 神秘・天啕|中|神秘関連イベント|預言者で特に影響大|
|⚔️ 軍備|中|戦術カード、兵力関連|試練直前で重要|

## ドローの流れ（1ターン）

1. **無料ドロー**: 6カテゴリから場の状況を反映して**3枚**提示
2. プレイヤーはその中から**1枚を選択**して取得
3. 🔥を消費して**追加ドロー**（より多くの選択肢を得る）
4. 🔥を消費して**マリガン**（提示された3枚をすべて引き直す）

## 3枚選出のルール

- 選出される3枚は**条件を満たしたカードの有無**で決まる
- **条件を満たしたカードがあれば候補**になる
- **条件を満たしていなければ候補にならない** (選出されない)
- 例: 場に平地がなければ、平地を条件とするカードは候補にならない

## 追加ドロー vs マリガン

- **追加ドロー** (🔥 1消費): さらに3枚提示され、そこからᵧ1枚選ぶ
- **マリガン** (🔥 2消費): 現在の提示を破棄して新たに3枚提示

マリガンはコストが高いが、「どうしても欲しいカテゴリが来ない」時の救溈手段として機能する。

## 災害カテゴリの特殊扱い

- 出現ウェイトは極めて低く設定
- 発生後、「○○のレガシー」を取得し、以後の同系災害発生率が低下（累積可能）
- 前兆イベントを導入予定（洪水・疫病・飢騲等）
- 敢えて災害を選ぶ選択肢も将来的に検討

## 序盤の工夫

- 初回行動ボーナス（プレイ毎に初回土地配置時などに🔥を得るなど）
- 🔥が多い時に発生しやすい隠しイベント（明文化は最小限に）
- 民からのクエスト（ランダムドローとは別に発生）

## 設計意図

完全なランダムではなく、「今場にある土地や特性が、次のドローの傾向を決める」仕組みにすることで、
プレイヤーが「この土地を取っておけば、次はこういうカードが来やすい」と予測・計画できるようにする。

## カードの視覚的表現（レアリティ表現）

カードのレアリティは、テキストではなく**視覚的に表現**することを基本方針とする。

### 表現方法

- **カードフレーム（枠）の変化**でレアリティを視角に伝える
- 正確な情報や詳細は**ツールチップ**で表示する

### フレーム変化の方向性（案）

- **銅（C）**: 粗い木枠・土っぽい質感
- **銀（UC）**: くすんだ金属枠・やや光沢のある質感
- **金（R）**: 金色の金属枠＋炎の模様が浮かび上がる
- **最上位（L）**: 金枠＋強く輝く炎のオーラや粒子エフェクト

### ツールチップでの表示例

カードにカーソルを合わせた際に、以下のような情報を表示する：

- レアリティ名
- 効果の詳細
- 出現条件
- 備考（例：特定の土地が存在する場合のみ出現 など）

### 土地カードの開発表現について

土地カードの開発回数による視角変化については、開発仕様（`03_development.md`）で別途定義する。  
レアリティ表現（フレーム変化）とは別軸で扱う。

### 目的

- プレイヤーがテキストを読まなくとも、一目でカードの価値や危険度を把握できるようにする
- 特に災害カードなどネガティブなカードは、フレームの印象で「危險そう」と直感的に伝わる設計を目指す

### 対象範囲

- イベントカードを中心にレアリティ表現を適用
- 土地カードについては開発表現を優先し、レアリティ表現は辅助的に扱う
